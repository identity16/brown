# Sankey Diagram (sankey-beta)

각 흐름의 **양(magnitude)** 이 띠 두께로 표시되는 흐름도. 에너지/예산/사용자 깔때기 등.

## 그리기 전에 물어볼 것 (AskUserQuestion)

1. **노드 = 무엇** — 흐름이 거쳐가는 단계/카테고리는 무엇인가? (예: 트래픽 소스 → 랜딩 → 가입 → 결제)
2. **흐름의 단위** — 사용자 수? 금액? 에너지? 단위를 정해두면 라벨이 일관된다.
3. **흐름 목록** — `source, target, value` 트리플의 리스트. 누락된 흐름은 없는지 (합이 안 맞으면 다이어그램이 이상해 보인다).
4. **노드 이름은 어떤 언어/형식으로** — 한국어/영어, 약어/풀네임.

값들의 합을 사전에 검증하면 좋다. 예: "랜딩 → 가입 + 랜딩 → 이탈"의 합이 "총 랜딩 유입"과 같아야 자연스럽다.

## 최소 문법

CSV 형식 (`source,target,value`):

```mermaid
---
config:
  sankey:
    showValues: true
---
sankey-beta

검색,랜딩,4200
소셜,랜딩,1800
이메일,랜딩,500
랜딩,가입,1800
랜딩,이탈,4700
가입,결제,900
가입,미결제,900
```

- 첫 빈 줄 다음부터 데이터.
- 노드는 따로 정의하지 않고 흐름에서 등장하는 이름이 곧 노드.

## 자주 하는 실수

- 흐름의 합이 안 맞음 → 띠가 어색하게 끊겨 보임. 입력 합 = 출력 합 검증.
- 순환(cycle)이 있는 그래프 → Sankey는 DAG에 적합. 순환이 필요하면 flowchart로.
- 단계가 너무 많아 띠가 가늘어지고 라벨이 겹침 → 핵심 단계로 줄여라.
